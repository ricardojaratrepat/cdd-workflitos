DETAIL · LIVE
01 · SOURCE
AAM / Hotjar
UX Signal · entrada del sistema
Escuchamos al cliente donde ya está hablando: grabaciones de sesión, encuestas in-app, eventos de frustración (rage clicks, dead clicks), tickets de soporte.
Trigger
webhook · tag “feedback”
Frecuencia
~120 / día
Latencia
~0 ms · push
02 · ORQUESTACIÓN
n8n
Workflow declarativo · sin código pegoteado
El webhook llega a n8n: normaliza payloads, deduplica, clasifica (bug / mejora / pregunta), enriquece con segmento, plan y canal — y enruta al siguiente agente.
Nodos
14 pasos
Branches
3 (bug / feat / ask)
Retries
auto · idempotente
03 · AGENTE · TICKETING
Agente Jira
Crea la tarjeta Launchpad lista para dev
Redacta título corto, contexto del cliente, criterios de aceptación, link a la sesión Hotjar y label de equipo. Adjunta evidencia. Asigna prioridad por impacto.
Output
Launchpad ticket
Campos
9 auto-llenos
Asignación
squad por code-owner
04 · AGENTE · DEV
Agente Dev
De ticket a Pull Request, sin intervención
Toma el Launchpad, levanta el repo correcto, escribe el código siguiendo convenciones del squad, agrega tests, abre PR con descripción, screenshots y checklist.
Tiempo medio
3-4 min
Tests
auto-generados
PRs / día
~30 · estimado

Google Chat

Notifica al squad en su canal habitual con resumen de cambios + link al PR. Un humano aprueba, comenta o pide ajustes. El agente itera. Listo para merge.
